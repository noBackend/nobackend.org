Go noBackend today!

A noBackend solution should provide a simple frontend API for typical backend tasks, but at the minimum it should provide methods for authentication and data persistence / synchronization.

Here is a loose list of some solutions we'd recommend you have a look at. A proper comparison table of existing noBackend solutions is coming soon.

  • deployd
  • Firebase
  • Hoodie
  • Kinvey
  • Parse
  • remoteStorage

Want to add another noBackend solution?

We get quite some suggestions for more providers or solutions to add to our list. First of all, please be patient, we give our best to keep up!

But before adding more solutions to the list, we kindly ask you to add an implementation of our example app, as well as a link to a proper documentation of the solutions frontend API.
